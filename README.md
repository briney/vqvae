# GCP-VQ-VAE: A PyTorch Implementation

This repository provides a clean PyTorch implementation of the **Geometry-Complete Protein Vector-Quantized Variational Autoencoder (GCP-VQVAE)**. This model allows for the tokenization (encoding) of protein backbone structures into discrete numerical representations and their subsequent reconstruction (decoding) back into 3D coordinates.

This work is based on the paper [**"Geometric-Complete VQ-VAE for Protein Structure"**](https://arxiv.org/abs/2402.10222).

## Features

- **GCPNet Encoder**: Utilizes an equivariant graph neural network with micro-steps and gating mechanisms to process protein geometry.
- **Vector Quantization**: Compresses the continuous latent space into a discrete set of tokens using a learned codebook.
- **Transformer Decoder**: A powerful transformer model learns to reconstruct the 3D structure from the sequence of discrete tokens.
- **6D Rotation Decoder**: Reconstructs the final backbone structure using rigid-body transformations (frames), with specialized loss functions for stable training.
- **Command-Line Interface**: Simple and effective CLI tools for training, evaluation, encoding, and decoding.


## Usage

This package provides a single command-line tool, `gpcvq`, with several subcommands: `train`, `eval`, `encode`, and `decode`.

A typical workflow involves:
1.  Training a new model on your own dataset using `gpcvq train`.
2.  Using the trained model checkpoint to encode your protein structures into tokens with `gpcvq encode`.
3.  Using the tokens to reconstruct the protein structure with `gpcvq decode`.

### Training

To train a model, you need to create a configuration YAML file that specifies the data paths, model architecture, and training parameters. You can use `src/gcpvqvae/configs/base.yaml` as a template.

1.  **Prepare your data:** The model expects a directory of PDB or mmCIF files.
2.  **Create a config file:** Copy and modify `base.yaml` to point to your data directory and set your desired hyperparameters.
3.  **Run the training script:**
    ```bash
    gpcvq train --config path/to/your_config.yaml --seed 42
    ```
    Checkpoints will be saved to the path specified in your config file (e.g., `runs/base_ckpt.pt`).

### Encoding

Once you have a trained model checkpoint, you can encode a specific chain from a PDB or mmCIF file into a sequence of discrete tokens.

```bash
gpcvq encode <input_path> --chain <chain_id> --out <output_path.npz> --checkpoint <model_checkpoint.pt>
```

**Example:**
```bash
# Encode chain A from the dummy PDB file using a trained model
gpcvq encode tests/data/dummy_A.pdb --chain A --out dummy_A_tokens.npz --checkpoint runs/base_ckpt.pt
```

This will create `dummy_A_tokens.npz`, which contains the token sequence and other necessary metadata for reconstruction.

### Decoding

You can use the `.npz` file generated by the encoder to reconstruct the protein backbone structure.

```bash
gpcvq decode <input.npz> --out <output_structure_file> --checkpoint <model_checkpoint.pt>
```

**Example:**
```bash
# Decode the tokens back into a PDB file
gpcvq decode dummy_A_tokens.npz --out dummy_A_decoded.pdb --checkpoint runs/base_ckpt.pt
```

This will create `dummy_A_decoded.pdb` containing the reconstructed 3D coordinates of the protein backbone.

### Evaluation

You can evaluate a trained model on a test dataset using a configuration file.

```bash
gpcvq eval --config path/to/your_eval_config.yaml --checkpoint <model_checkpoint.pt>
```

## Configuration

The model and training process are controlled by a YAML configuration file. The `src/gcpvqvae/configs/base.yaml` file provides a starting point with parameters based on the original paper.

The main sections are:
-   `data`: Specifies the data path, sequence length cap, and other data processing parameters.
-   `model`: Defines the architecture of the GCPNet encoder, transformer, VQ codebook, and decoder.
-   `train`: Contains training hyperparameters like learning rate, batch size, number of steps, and checkpoint save path.
